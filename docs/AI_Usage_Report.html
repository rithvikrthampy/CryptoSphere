AI Assistance & Build Report — Crypto Sphere

Project: Responsive crypto prices SPA using CoinGecko • Stack: React, Vite, TypeScript, Tailwind, React Query, Chart.js • Deployment: Vercel
Build Note: This project was built primarily via AI prompting; very minimal manual coding was required.

1) Executive Summary

Crypto Sphere is a responsive single‑page application showing live cryptocurrency data from CoinGecko. It features a homepage markets table (Price, 1h, 24h, 7d, 24h Volume, Market Cap, 7‑day sparkline), a prominently featured VANRY/USDT card, interactive coin detail charts with multiple ranges, search/filters, and a dark/light theme. The build combined AI-assisted scaffolding and component generation with targeted manual work for configuration and deployment.

2) Tools & Roles

AI Tools

  • GPT Codex — initial project skeleton and structure decisions.
  • Claude Code CLI — code generation and optimization passes for components and flows.
  • Codex — README creation, code analysis, API proxy logic and rate‑limit mitigations.

Platforms

  • CoinGecko — free API for markets, detail, and charts.
  • Vercel — serverless proxy + static hosting (production).
  • GitHub — source control and CI entry point for Vercel.

3) What AI Produced

  • Vite + React + TypeScript scaffold and Tailwind setup; routing and React Query boilerplate.
  • API layer with a Vercel serverless proxy to keep the CoinGecko key private; dev proxy in Vite.
  • Core screens and components:
    • Homepage with featured VANRY pair and markets table (required columns).
    • Search bar, quick filters, pagination, and sparkline rendering.
    • Coin detail page with chart ranges (24H, 7D, 1M, 3M, 1Y, Max).
    • Dark/light theme toggle, skeleton loaders, and UI animations.
    • Market overview sidebar (global stats, trending, gainers/losers).
  • README authoring and troubleshooting guidance (rate limits, Pro vs free base URL).
  • Code quality passes: types, formatting helpers, and error handling for 429/404/5xx.

4) Manual Interventions

  • Created a CoinGecko API key and provisioned environment variables:
COINGECKO_API_KEY=...
COINGECKO_USE_PRO=false
  • Set up .env locally and configured Vercel Project Env Vars.
  • Resolved Vercel deployment issues by prompting and applying config fixes (proxy routing, SPA fallback).
  • Pushed code to GitHub and connected the repo to Vercel for automatic builds and deploys.
  • Tuned rate‑limit behavior: reduced refresh intervals, capped markets to 1,000 coins, limited sidebar lists to 3 items.

5) Architecture Overview

  • UI: React + Tailwind, Framer Motion animations, dark/light theme via class toggle + localStorage.
  • Data: TanStack Query with caching and background refresh; compact formatters for currency and numbers.
  • Charts: Chart.js via react‑chartjs‑2; row sparklines via lightweight inline SVG.
  • Proxy: Vercel function at /api/coingecko/* with optional Pro header and s‑maxage caching.
  • Dev: Vite proxy forwards to correct CoinGecko base (free vs pro) based on env flags.
